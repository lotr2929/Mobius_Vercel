Quick Start

Getting up and running

Follow these five steps in order. By the end, Mobius will be installed, connected to your Google account, and ready to use.

1
Open Mobius in your browser

Go to mobius-vercel.vercel.app and log in with your username and password.

2
Install it as an app (recommended)

On Android: In Chrome, tap the three-dot menu and select Add to Home Screen.
On desktop: Click the install icon in the address bar in Edge or Chrome.

3
Connect your Google account

Mobius connects your personal Google account automatically on first login. If this did not happen, type Google: connect personal and press Ask. You will be redirected to Google to sign in and approve access. Mobius returns automatically afterwards.

4
Check the startup panel

Each time Mobius loads, a startup panel appears at the top of the chat showing the status of Google accounts, Dropbox, Local AI, and the last sync. All items showing a tick means you are fully connected.

5
Ask your first question

Type anything in the input box and press Ask. Mobius builds a query and shows you a preview of what it will send. Press Ask a second time to send it and receive a response.

Try asking: What are my upcoming calendar events this week?

Once connected, you can add more Google accounts, connect Dropbox, load files, run personal projects, or start a coding session. Each is covered in its own module in Section 2.

Two shortcuts worth knowing from the start

Every command group in Mobius follows a consistent pattern:

You typeWhat you getExample
Command: (no argument)A status summary of what is currently activeGoogle: shows connected accounts
Command?A help card listing all sub-commandsGoogle? shows all Google: commands

If you are ever unsure what a command can do, add ? and press Ask.

Section 1 — Introducing Mobius

1. What is Mobius?

Mobius is a personal AI assistant that runs in your web browser. It connects your Google accounts, Dropbox, files, and projects into one place, and routes your questions to whichever AI model is most appropriate. It can be installed on your phone and desktop like a regular app.

At its core, Mobius coordinates four things:

  • AI models — Mobius routes your questions to the best model for the task. You can choose a model explicitly, or let Mobius decide. If one fails, it tries the next automatically.
  • Your accounts — Google (up to three accounts) and Dropbox. Mobius reads your calendars, emails, and file listings so the AI always has the right context when you ask about them.
  • Your projects — a personal journal, a study programme, a software project — Mobius loads the relevant file and keeps the AI informed throughout the session.
  • Your memory — Mobius maintains a personal rulebook of your preferences and working style. It applies these automatically to every query, on every device.

How Mobius handles a request

Mobius always tries the simplest, most reliable approach first:

  1. Commands first. If you type a command such as Sync: all or Google:, Mobius handles it immediately with no AI involved.
  2. Your rules next. Before sending anything to an AI, Mobius reads your personal rulebook so the model already knows your preferences.
  3. AI when needed. Only when a command alone cannot answer your question does Mobius call an AI model.
  4. Check the answer. After every AI response, Mobius checks for signs of trouble — apologies, uncertainty, truncation — and flags anything worth reviewing.

Modules

Mobius comes with two free modules — Personal and Projects. Additional modules can be added as your needs grow. Section 2 covers each one in detail.

Mobius works on any device. Local file access and local AI models are desktop-only features.
Section 1 — Introducing Mobius

2. Asking Questions

How asking works

Mobius uses a two-step process so you can see exactly what will be sent before it is sent.

1
First press of Ask: Type your question and press Ask. Mobius assembles a query and shows you a preview box — the AI model selected, any context files attached, and a count of history messages included. Review it before continuing.
2
Second press of Ask: Press Ask again to send the query and receive a response. Press Escape at any point to cancel.
On desktop, Ctrl+Enter works the same as clicking the Ask button.

Choosing a model

By default, Mobius uses Groq. To choose a different model, begin your message with Ask: modelname. Mobius remembers your choice for the rest of the session.

CommandBest for
Ask: GroqFast everyday questions (default)
Ask: GeminiLong or detailed answers, documents, and images
Ask: MistralCoding tasks
Ask: GitHubGPT-4o via GitHub Models
Ask: WebQuestions that need current information from the internet
Ask: QwenPrivate coding — runs entirely on your device
Ask: DeepSeekPrivate reasoning — runs entirely on your device
Qwen and DeepSeek require Ollama to be running on your desktop computer. They are not available on mobile.

Attaching files

Press the + button to attach files. Images are routed to Gemini for visual analysis. Text documents are included as readable content. You can attach multiple files; press the remove button on any chip to detach one.

What Mobius includes automatically

Every query includes two things that the AI sees but you do not need to provide:

  • Your environment — the current date, time, location, device type, and browser.
  • Your rulebook — your preferences, working rules, and current project state from mobius.json. See page 4 for details.

Flagged responses

After every AI response, Mobius checks for common problems — apologies, expressions of uncertainty, truncated output. If any are detected, an amber warning bar appears below the response. Use Remember: to record a lesson from the exchange if needed.

Section 1 — Introducing Mobius

3. Mobius Memory

Mobius maintains a personal rulebook that records your preferences, your working style, corrections, and your current project state. This is stored in a file called mobius.json in your Google Drive Mobius folder, and it is included automatically in every AI query.

The practical effect is that you never have to repeat yourself. Instructions like “always use British English” or “never suggest jQuery” are applied in every session, on every device, without you having to type them again.

Adding a rule

Use Remember: whenever you want Mobius to retain something. You can direct a rule to a specific section by adding a prefix:

  • Remember: always wrap fetch calls in try/catch — added to rules
  • Remember: preference I prefer concise answers with examples — added to preferences
  • Remember: correction deploy uses deploy.bat, never manual git — added to corrections
  • Remember: do not suggest jQuery — added to the do-not list

The entry is saved to Drive immediately and applied from the next query onwards.

Viewing and editing rules

  • Mobius: — shows session duration, flags raised, and rule counts
  • Mobius? — lists all Mobius memory commands
  • Mobius: all — full numbered list of every rule
  • Forget: 7 — removes rule number 7
  • Amend: 7 updated text — replaces rule 7 with new text
  • Mobius: state — shows the current project focus and next steps

End-of-session review

Type Review: at the end of a working session. Mobius presents any flags raised during the session as proposed lessons. Accept the ones worth keeping, skip the rest, or add your own before saving everything back to Drive.

Keeping the rulebook tidy

Use Mobius: refine to ask Gemini to review your rulebook. It checks for duplicates, contradictions, and vague rules, and proposes improvements five at a time. Each proposed change requires your approval before anything is saved.

Nothing is written to your rulebook without your explicit approval. Every lesson is confirmed individually before it is saved.

Personal Free

The Personal module connects Mobius to your email, calendar, Google Drive, and Dropbox. Once your accounts are connected, Mobius can draw on your own data when answering questions, keep indexes up to date automatically, and give you a single place to check what is happening across your accounts.

What the Personal module covers
  • Asking about upcoming calendar events across all connected Google accounts
  • Checking unread emails without opening Gmail
  • Searching your Google Drive and Dropbox by name or keyword
  • Connecting up to three Google accounts (personal, family, work)
  • Automatic background sync to keep everything current

Pages in this module

  • Connecting Your Accounts — step-by-step instructions for Google and Dropbox
  • Your Files — the Mobius Drive folder, loading files, and local folder access
  • Keeping in Sync — how sync works, how to check it, and how to fix problems
Mobius is read-only. It reads your calendars, emails, and file listings, but it never sends email, creates calendar events, or modifies anything outside the Mobius Drive folder.
Personal Module

Connecting Your Accounts

Mobius connects to Google and Dropbox using OAuth — a standard authorisation process where you sign in directly with the service and grant Mobius read access. Your passwords are never stored or seen by Mobius.

Google accounts

Mobius supports up to three Google accounts at the same time, labelled personal, family, and work.

Connecting your personal account

1
Type Google: connect personal and press Ask. This also runs automatically on your first login.
2
You are redirected to Google. Sign in with your personal Gmail address.
3
Google asks whether to allow Mobius access. Select Allow.
4
You return to Mobius automatically. The startup panel will confirm the connection on next load.

Connecting additional accounts

  • Google: connect family — for a shared family Gmail
  • Google: connect work — for a work Google account
When Google redirects you to sign in, make sure you select the correct account for each label.

Checking and managing connections

  • Google: — shows which accounts are currently connected
  • Google? — lists all Google: commands
  • Google: disconnect personal — removes a connected account

What Mobius reads from Google

DataWhat Mobius readsWhat Mobius never does
CalendarEvent titles, times, and locations for the next 60 daysNever creates, edits, or deletes events
GmailUnread mail only: sender, subject, and date. No message body.Never reads message content. Never sends email.
DriveFile and folder names with last-modified datesNever modifies files outside the Mobius folder

Dropbox

1
Type Dropbox: connect and press Ask.
2
You are redirected to Dropbox. Sign in and click Allow.
3
You return to Mobius. Type Sync: dropbox to build the initial file index.

Checking Dropbox

  • Dropbox: — shows connection status and total file count
  • Dropbox? — lists all Dropbox: commands
  • Dropbox: list — lists the root of your Dropbox
Mobius has read-only access to Dropbox. It never modifies, moves, or deletes files.

Troubleshooting connections

If a connection shows an error at startup, reconnect it by typing Google: connect personal. This refreshes the access token and usually resolves the problem immediately.
  • On mobile, if Mobius does not return automatically after a Google redirect, navigate back manually. Your connection will have been saved.
  • If the startup panel reports a failing service, type Status for a full live health check.

Other services

Support for Outlook, Yahoo Mail, and other services is planned for a future update.

Personal Module

Your Files

The Mobius Drive folder

Mobius maintains a single folder called Mobius in your personal Google Drive. This is the only location Mobius writes to. It contains:

  • Sync index files: calendar.index, email.index, drive.index, dropbox.index
  • mobius.json — your personal rulebook and memory
  • Working copies of files loaded with Focus:
  • Output files from Code sessions
Install Google Drive for Desktop so the Mobius folder — including mobius.json — is always available on your desktop and accessible from your phone.

Loading a Drive file with Focus:

Use Focus: to load any Google Drive file into your current session. Once loaded, the file is attached to every AI query automatically until you detach it. The AI reads the file content before each response without you having to paste anything.

CommandWhat it does
Focus: filenameFinds and loads a Drive file. If no file matches, creates one in the Mobius folder.
Focus: add textAppends a timestamped entry to the loaded file and saves to Drive.
Focus: updateWrites current session changes back to the original Drive file.
Focus: endDetaches the file from the session.
Focus:Shows the currently loaded file.
Focus?Lists all Focus: commands.

Local folders (desktop only)

On desktop Chrome or Edge, Mobius can read folders on your local hard drive. This gives the AI access to files that are not on Drive.

CommandWhat it does
AccessOpens a folder picker. Grant read access to a local folder. This is saved and remembered across sessions.
ListShows the contents of the granted folder.
Find: filenameSearches by name. Add Ext: pdf, From: last month, or To: today to narrow results.
Local file access is read-only. Mobius never modifies or deletes local files. This feature is not available on mobile.
Personal Module

Keeping in Sync

Mobius stores lightweight snapshots of your account data as index files in your Mobius Drive folder. When you ask about your calendar, emails, or files, Mobius reads from these indexes rather than querying Google or Dropbox live. This makes responses faster and means things still work if a service is temporarily unavailable.

What gets synced

FileContents
calendar.indexUpcoming events from all connected calendars — title, date, time, and location
email.indexUnread email metadata from all inboxes — sender, subject, and date
drive.indexFile listings from all connected Drives — name and last-modified date
dropbox.indexAll Dropbox files — name, path, size, and last-modified date

When sync runs

  • On every login — a full sync runs automatically in the background.
  • Every five minutes — an incremental Google sync runs while Mobius is open.
  • Dropbox — syncs on login and on demand only.

Checking sync status

  • The sync indicator next to your username shows the last sync time. A warning symbol means something went wrong.
  • Sync: — shows last sync times for all services at a glance.
  • Sync? — lists all Sync: commands.
  • Sync: status — full timestamp table for every connected account.

Syncing manually

CommandWhat it does
Sync: allFull refresh — all Google accounts and Dropbox
Sync: calendarsCalendars only
Sync: emailsEmail metadata only
Sync: driveDrive file listings only
Sync: dropboxDropbox only

Fixing sync problems

Most sync failures are caused by an expired Google authorisation token. Type Google: connect personal to refresh it, then run Sync: all.
  • If one account fails but others are fine, reconnect just that account: for example, Google: connect work.
  • For Dropbox failures: type Dropbox: connect, then Sync: dropbox.
  • Type Status for a full live health check across all connected services.
Section 2 — Modules

Projects Free

The Projects module provides a structured way to work on ongoing personal projects — a daily practice, a study programme, a journal, or any activity you return to regularly. Mobius stores the project file on your Google Drive and loads it automatically so every session continues from where the last one ended.

How a Mobius project works
  • A dedicated text file on Google Drive that Mobius loads at the start of each session
  • A cumulative record — entries, notes, and responses collected in one place
  • The AI reads the full project history before every response
  • New entries are appended and saved to Drive in a single command

Setting up a project

1
Create a text or markdown file on Google Drive for your project. Open it and write a short description of what the project is — the AI reads this every session.
2
In Mobius, type Focus: your-project-name. Mobius finds the file and loads it.
3
Ask questions or request analysis. The AI reads your project file automatically before responding.
4
To add a new entry: type Focus: add your note here. Mobius appends it with a timestamp and saves immediately to Drive.
5
When the session is done, type Focus: end. Everything is already saved. Next session, load the file again and carry on.

Key commands

CommandWhat it does
Focus: filenameLoads a project file from Drive
Focus: add [text]Appends a timestamped entry and saves to Drive
Focus: updateWrites session changes back to the original Drive file
Focus: endDetaches the file from the session
Focus:Shows the currently loaded file
Keep project files in your Mobius Drive folder for fastest access, and use a consistent filename so Focus: finds it immediately each session.
Projects Module

A Typical Project Session

Opening the session

Type Focus: evening-grace. Mobius locates the file on Drive and loads it. A confirmation message shows the filename and confirms it is attached to the session.

Asking questions in context

Once the file is loaded, ask anything you like. The AI reads your project history before responding, so it already knows what you have done and does not need to be briefed each time.

  • What are the themes from last week's entries?
  • Summarise what I have covered so far.
  • What questions have come up that I have not addressed yet?

Adding a new entry

Type: Focus: add Today I studied the concept of grace in Romans 5. Key insight: Paul distinguishes between grace as undeserved favour and grace as transforming power.

Mobius appends the text with the current date and time and saves the file to Drive immediately.

Asking for a summary

At the end of a session, ask: Give me a brief summary of what we covered today. Once you have the response, use Focus: add to save the summary as your final entry for the session.

Closing the session

Type Focus: end. All entries are already saved to Drive. The next time you load the file, everything is there.

Practical notes

  • Begin each project file with a short paragraph describing what the project is. The AI reads this at the start of every session and immediately understands the context.
  • Use Focus: add frequently — it is the fastest way to record a thought without leaving Mobius.
  • You can maintain multiple project files. Type Focus: end before loading a different one.
  • To check what is currently loaded at any point, type Focus:.

Section 2 — Modules

Coding Add-on

The Coding module provides a structured AI coding assistant. Rather than pasting code into a chat window, Code Mode first gives the AI a verified, accurate picture of your project — what functions exist, how they connect, what dependencies are declared — before you ask it anything. This reduces errors and means the AI does not have to guess at your project structure.

What Code Mode provides
  • An index of every function, route, import, and export in your project
  • Automatic static analysis to catch common mistakes — no AI involved, instant results
  • An architectural summary generated by Gemini
  • A structured audit with a prioritised list of issues to address
  • All output saved automatically to your Google Drive Mobius folder

Starting a coding session

1
Type Code: projectname and press Ask. A folder picker opens.
2
Navigate to the folder containing your project and select it.
3
Mobius loads the project. A badge showing the project name appears in the header. All AI queries for the rest of the session use your project as context.

The analysis pipeline

CommandWhat it doesOutput file
Code: repoIndexes all functions, imports, exports, and routes nAI.repo
Code: scanStatic analysis — no AI, instant nAI.scan
Code: mapGemini architectural summary.map
Code: audit newFull audit — scan results plus Gemini fix plan.audit
Code: auditResume an existing auditUpdated .audit
Code: allRuns repo, map, and audit in sequenceAll three
Code: statusSaves a snapshot of what has been done and what remains.status
Code: endEnds the session—

To check session status at any point, type Code:. To see all available commands, type Code?.

Practical notes

  • Run Code: all at the start of each new session for a complete, up-to-date overview of the project.
  • Code: scan involves no AI and produces no guesswork. Run it frequently as a reliable baseline check.
  • Always ask the AI to describe any proposed change before applying it. Do not apply code you have not reviewed.
Code Mode requires desktop Chrome or Edge, as it uses local file access to read your project. It is not available on mobile.

For example: Google: shows connected accounts. Google? lists all Google: commands. Code: shows the active session. Code? lists all Code: commands.

Useful habits

  • Read the query preview before pressing Ask the second time. It shows exactly what Mobius will send, including which context files are attached.
  • For sensitive tasks, use Ask: Qwen or Ask: DeepSeek. These models run entirely on your device and send nothing to external servers.
  • Use New: topic to start a fresh session and ask the first question in a single step.
  • Right-click a chat history entry on desktop, or long-press on mobile, for copy and context options.
  • Type Review: at the end of any productive session to review flags and save lessons.
  • Run Mobius: refine after a session in which you have added three or more rules.
  • Type Mobius: state at the start of a session to see where you left off on your current project.
  • After connecting a new account, run Sync: all to build the indexes immediately.
  • If the startup panel shows a service as disconnected, reconnect it before starting work.
  • Type Status if something seems wrong — it runs a live check across all connected services.
  • In a coding session, run Code: all at the start for a complete, up-to-date overview of the project.
  • Install Google Drive for Desktop so mobius.json is kept in sync between your desktop and other devices automatically.
