Quick Start

Get up and running in 5 minutes

Follow these steps in order. You'll have Mobius installed, connected, and ready to use.

1
Open Mobius in your browser

Go to mobius-vercel.vercel.app and log in with your username and password.

2
Install it as an app (recommended)

On Android: In Chrome, tap the three-dot menu → Add to Home Screen.
On desktop: Click the install icon (⊕) in the address bar in Edge or Chrome.

3
Connect your Google account

This happens automatically on first login. If it didn't, type Google: connect personal and press Ask. You'll be redirected to Google — sign in and tap Allow. Mobius returns automatically.

4
Check the startup panel

At the top of the chat you'll see a startup summary — Google accounts, Dropbox, Local AI, and sync status. Everything green means you're fully connected.

5
Ask your first question

Type anything in the input box and press Ask. Mobius shows you a preview of what it will send — press Ask again to get the response.

Try: "What are my upcoming calendar events this week?"

You're in. For more — connecting a second Google account or Dropbox, managing files, personal projects, or coding — read the relevant module in Section 2.

The golden rule

Every command group has two helper shortcuts you can always type:

If you're ever unsure what a command can do, add ? and press Ask.

Section 1 — Introducing Mobius

1. What is Mobius?

Mobius is your personal AI assistant — a single place where you can ask questions, manage your accounts and files, and work on your projects. It runs in any web browser and installs on your phone and desktop like a regular app.

Think of it as a smart coordinator that connects everything you use:

  • AI models — Mobius routes your questions to the best AI. You can choose, or let Mobius decide. If one fails, it automatically tries the next.
  • Your accounts — Google (up to three accounts) and Dropbox. Mobius reads your calendars, emails, and files so the AI always has the right context.
  • Your projects — a personal journal, a study practice, a software project — Mobius keeps everything connected.
  • Your memory — Mobius remembers your preferences and working style. You never have to repeat yourself.

How it works

Mobius always tries the simplest, most reliable approach first:

  1. Commands first — if you type a command like Sync: all or Google:, Mobius handles it instantly with no AI involved.
  2. Your rules next — before asking an AI anything, Mobius reads your personal rulebook so the AI knows your preferences.
  3. AI when needed — only when a command can't answer your question does Mobius call an AI model.
  4. Check the answer — after every AI response, Mobius automatically checks for errors or uncertainty and flags anything suspicious.

Modules

Mobius comes with two free modules — Personal and Projects. Additional modules can be added as your needs grow. See Section 2 for each module.

Mobius works on any device. Some features — local file access, local AI — are desktop-only.
Section 1 — Introducing Mobius

2. Asking Questions

The two-step process

Mobius shows you exactly what it will send before it sends it — keeping you in control.

1
First Ask: Type your question and press Ask. Mobius builds a query package and shows you a preview — which AI model, which context files, what attachments. Review it.
2
Second Ask: Press Ask again to send. Press Escape at any point to cancel.
Ctrl+Enter works the same as clicking Ask on desktop.

Choosing a model

Type Ask: modelname before your question to choose a specific AI. Mobius remembers your choice.

CommandBest for
Ask: GroqFast everyday questions (default)
Ask: GeminiLong or detailed answers, documents, images
Ask: MistralCoding
Ask: GitHubGPT-4o alternative
Ask: WebQuestions needing current information
Ask: QwenPrivate coding — runs on your device only
Ask: DeepSeekPrivate reasoning — runs on your device only
Local models (Qwen, DeepSeek) require Ollama running on your desktop. Not available on mobile.

Attaching files

Tap + to attach files. Images go to Gemini for visual analysis. Documents are included as text. Multiple files can be attached — tap ✕ to remove.

What Mobius always includes

Every query automatically includes two things the AI sees but you don't need to type:

  • Environment — current date, time, location, device, browser.
  • Your rulebook — your preferences, rules, and project context from mobius.json. See page 4.

Flagged responses

After every AI response, Mobius checks for signs of trouble — apologies, uncertainty, truncation. If anything is flagged, an amber ⚠️ bar appears below the response. Use Remember: to capture a lesson from it.

Section 1 — Introducing Mobius

3. Mobius Memory

Mobius remembers things about you — your preferences, your working style, mistakes to avoid, and where you are in your current project. This memory is stored in a file called mobius.json in your Google Drive Mobius folder, and it is included in every AI query automatically.

You never have to repeat yourself. If you've told Mobius "always use British English" or "never suggest jQuery", it applies that every time — in every session, on every device.

Teaching Mobius

Whenever Mobius does something wrong, or you want it to remember something:

  • Remember: always wrap fetch calls in try/catch
  • Remember: preference I prefer concise answers with examples
  • Remember: correction deploy uses deploy.bat, never manual git
  • Remember: do not suggest jQuery

The lesson is saved to Drive immediately and applied from the next query onwards.

Viewing and editing rules

  • Mobius: — session pulse: how long running, flags raised, rule counts
  • Mobius? — help card: all Mobius memory commands listed
  • Mobius: all — full numbered list of everything Mobius knows
  • Forget: 7 — removes rule number 7
  • Amend: 7 updated text — replaces rule 7
  • Mobius: state — current project focus and next steps

End-of-session review

Type Review: at the end of a session. Mobius presents any flags as proposed lessons — tick the ones worth keeping, skip the rest, add your own. Saved to Drive.

Keeping rules clean

Use Mobius: refine to ask Gemini to review your rulebook — it flags duplicates, contradictions, and vague rules, and proposes improvements five at a time. You approve each change.

Nothing is ever saved to Mobius's memory without your explicit approval. Every lesson is individually confirmed.
Section 2 — Modules

Personal Free

The Personal module connects Mobius to your digital life — email, calendar, Drive, and Dropbox. Once connected, Mobius can answer questions using your own data, keep everything in sync, and give you a single place to manage it all.

What you can do
  • Ask about upcoming calendar events across all your Google accounts
  • Check unread emails without opening Gmail
  • Search your Google Drive and Dropbox
  • Connect up to three Google accounts (personal, family, work)
  • Keep all data synced and up to date automatically

Pages in this module

  • Connecting Your Accounts — step-by-step for Google and Dropbox
  • Your Files — Drive, Mobius folder, local folders
  • Keeping in Sync — how sync works, how to check it, how to fix it
Mobius is read-only. It reads your calendars, emails, and file listings — but never sends emails, creates events, or modifies anything outside the Mobius Drive folder.
Personal Module

Connecting Your Accounts

Mobius connects via a secure sign-in process called OAuth — you log in directly with Google or Dropbox, and Mobius receives a token to access your data. Your passwords are never stored by Mobius.

Google accounts

Mobius supports three Google accounts simultaneously, labelled personal, family, and work.

Connecting your personal account

1
Type Google: connect personal and press Ask. (This also happens automatically on first login.)
2
You are redirected to Google's sign-in page. Sign in with your personal Gmail.
3
Google asks you to allow Mobius access. Tap Allow.
4
You return to Mobius automatically. The startup panel will show ✅ personal on next load.

Connecting family or work accounts

  • Google: connect family — for your shared family Gmail
  • Google: connect work — for your work email
When prompted by Google, make sure you sign in with the correct account for each label.

Checking and managing Google connections

  • Google: — instant status: which accounts are connected
  • Google? — help card: all Google: commands
  • Google: disconnect personal — remove a connected account

What Mobius reads from Google

DataWhat Mobius readsWhat Mobius never does
CalendarEvent titles, times, locations — next 60 daysNever creates, edits or deletes events
GmailUnread email — sender, subject, date only. No body content.Never reads content. Never sends email.
DriveFile and folder names, modified datesNever modifies files outside Mobius folder

Dropbox

1
Type Dropbox: connect and press Ask.
2
You are redirected to Dropbox. Sign in and click Allow.
3
You return to Mobius. Type Sync: dropbox to build the initial file index.

Checking Dropbox

  • Dropbox: — connection status and file count
  • Dropbox? — help card: all Dropbox: commands
  • Dropbox: list — list the root of your Dropbox
Mobius has read-only access to Dropbox. It never modifies, moves, or deletes any file.

Troubleshooting

If a connection shows an error at startup, reconnect it: Google: connect personal. This refreshes the access token and usually resolves the issue immediately.
  • On mobile, if you're redirected away and Mobius doesn't return automatically, navigate back manually — your connection will still be saved.
  • If the startup panel shows a service failing, type Status for a full live diagnosis.

Other services

Support for Yahoo Mail, Outlook, and other services is planned for a future update.

Personal Module

Your Files

The Mobius Drive folder

Mobius maintains a single folder called Mobius in your personal Google Drive. This is the only place Mobius writes to. It contains:

  • Sync index files — calendar.index, email.index, drive.index, dropbox.index
  • mobius.json — your personal rulebook and memory
  • Working copies of files you've loaded with Focus:
  • Output files from Code Mode sessions
Install Google Drive for Desktop on your HP and phone so the Mobius folder — including mobius.json — is always available offline on both devices.

Loading a Drive file — Focus:

Use Focus: to load any Google Drive file into your session. The file is attached to every AI query until you detach it — the AI reads it automatically before responding.

Personal Module

Keeping in Sync

Mobius keeps lightweight snapshots of your account data as index files in your Mobius Drive folder. When you ask about your calendar or emails, Mobius uses these indexes — making responses faster and working even when services are temporarily unavailable.

What gets synced

FileContains
calendar.indexUpcoming events from all connected calendars — title, date, time, location
email.indexUnread email metadata from all inboxes — sender, subject, date
drive.indexFile listings from all connected Drives — name, modified date
dropbox.indexAll Dropbox files — name, path, size, modified date

When sync runs

  • On every login — full sync runs automatically in the background.
  • Every 5 minutes — incremental Google sync while Mobius is open.
  • Dropbox — on login and on demand only.

Checking sync

  • The sync indicator next to your username: ✓ HH:MM = synced, ⟳ = syncing, ⚠ = problem.
  • Sync: — last sync times for all services at a glance.
  • Sync? — help card: all Sync: commands.
  • Sync: status — full timestamp table for all accounts.

Syncing manually

CommandWhat it does
Sync: allFull refresh — all Google + Dropbox
Sync: calendarsCalendars only
Sync: emailsEmail metadata only
Sync: driveDrive listings only
Sync: dropboxDropbox only

Fixing sync problems

Most sync failures are caused by an expired Google connection. Try: Google: connect personal to refresh, then Sync: all.
  • If one account fails but others don't, reconnect that specific account: Google: connect work.
  • For Dropbox failures: Dropbox: connect then Sync: dropbox.
  • Type Status for a full live health check of all systems.
Section 2 — Modules

Projects Free

The Projects module gives you a structured way to manage ongoing personal projects — a daily practice, a study programme, a journal, or any activity you return to regularly. Mobius keeps the project file on your Google Drive and connects it to the AI so every session picks up where you left off.

What makes a Mobius project
  • A dedicated file on Google Drive that Mobius loads at the start of each session
  • A cumulative record — entries, notes, and AI responses all in one place
  • AI that reads your full project history before answering
  • The ability to add new entries and save everything back to Drive instantly

Starting a project

1
Create a file on Google Drive for your project. Start it with a brief description of what the project is — the AI reads this first.
2
In Mobius, type Focus: your-project-name. Mobius finds the file and loads it.
3
Ask questions or request analysis. The AI reads your file automatically.
4
Add a new entry: Focus: add your note here. Mobius appends it with a timestamp and saves to Drive.
5
When done: Focus: end. Next session, just load the file again — it's all there.

Key commands

CommandWhat it does
Focus: filenameLoad a project file from Drive
Focus: add [text]Add a timestamped entry and save
Focus: updateWrite back to the original Drive file
Focus: endDetach the file
Focus:Check what's currently loaded
Keep project files in your Mobius Drive folder for the fastest access. Use a consistent filename so Focus: finds it immediately every session.
Projects Module

A Typical Project Session

Opening the session

Type Focus: evening-grace. Mobius finds the file and loads it:

🟢 Focused on: evening-grace.md
📝 Current content: [your previous entries]
File will be attached to all AI queries this session.

Asking questions in context

Ask anything — Mobius has already loaded your project history:

  • "What are the themes from last week's entries?"
  • "Summarise what I've covered so far."
  • "What questions have come up that I haven't addressed yet?"

The AI reads your file before answering — it knows your history without you repeating it.

Adding a new entry

Type: Focus: add Today I studied the concept of grace in Romans 5. Key insight: Paul distinguishes between grace as undeserved favour and grace as transforming power.

Mobius appends this with today's date and time and saves to Drive immediately.

Getting a summary

Ask: "Give me a brief summary of today's session." Then use Focus: add to save the AI's summary as your final entry.

Closing the session

Type Focus: end. Your entries are already saved to Drive.

Tips

  • Start each project file with a paragraph describing the project — the AI reads it immediately and understands context.
  • Use Focus: add liberally — it's a fast way to capture thoughts without leaving Mobius.
  • You can have multiple projects — just Focus: end before loading a different file.
  • Check what's loaded at any time: type Focus:.

Section 2 — Modules

Coding Add-on

The Coding module gives you a structured AI coding assistant. Rather than pasting code into a chat window, Code Mode gives the AI verified, accurate knowledge of your project before it suggests anything — reducing errors and saving time.

What Code Mode does
  • Reads your project and builds an index of every function, route, and dependency
  • Checks for common mistakes automatically — no AI guesswork
  • Asks Gemini for an architectural overview of your project
  • Runs a structured audit with a prioritised fix plan
  • Saves all output to your Google Drive Mobius folder automatically

Starting a session

1
Type Code: projectname. A folder picker opens.
2
Navigate to the parent folder containing your project and select it.
3
Mobius loads the project. A green ⌨ projectname badge appears in the header. All AI queries now use your project as context.

The pipeline

CommandWhat it doesOutput
Code: repoMaps all functions, imports, exports, routes nAI.repo
Code: scanChecks for mistakes — no AI, instant nAI.scan
Code: mapGemini architectural summary.map
Code: audit newScan + Gemini briefing and fix plan.audit
Code: auditResume existing auditUpdated .audit
Code: allrepo → map → audit in one stepAll three
Code: statusSnapshot what's been done and what's next.status
Code: endEnd the session—

Check session status anytime: type Code:. See all commands: type Code?.

Tips

  • Run Code: all at the start of each session for a complete fresh overview.
  • Code: scan is entirely automatic — no AI, no hallucinations. Run it often.
  • Always ask the AI to describe changes before applying them. Never apply code you haven't reviewed.
Code Mode requires desktop Chrome or Edge — it uses local file access to read your project files. Not available on mobile.
