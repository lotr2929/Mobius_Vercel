1. Overview

Mobius is a personal AI assistant and coordinator deployed as a Progressive Web App (PWA). It routes queries across multiple AI models, integrates with Google Workspace and Dropbox, and includes a structured coding assistant framework.

Mobius is designed around three principles:

  • Manual control — you review every query before it goes to an AI.
  • Need-to-know context — only relevant information is sent to the AI, not your entire history.
  • Read-only everywhere — Mobius reads from your accounts but only writes to its own dedicated folder in your personal Google Drive.

Mobius runs in the browser and can be installed on desktop and mobile as a standalone app.

2. Accessing Mobius

  • Production: https://mobius-vercel.vercel.app
  • Stable: https://mobius-plum.vercel.app
  • Login: Enter your username and password on the login screen.
  • PWA Install: On Chrome or Edge, click the install icon in the address bar to install Mobius as a desktop or mobile app.

First Login

On first login, Mobius will automatically prompt you to connect your personal Google account. After connecting, it will begin syncing your calendars, emails, and Drive file listings in the background.

To connect additional Google accounts (family, work) or Dropbox, use the Google: connect and Sync: dropbox commands.

3. Interface

The Mobius interface has three main areas:

  • Header — Shows the Mobius logo, your username, a sync status indicator (✓ HH:MM when synced, ⟳ syncing, ⚠ offline), and Log Out button.
  • Input area — The text box where you type queries or commands. Supports ↑/↓ arrow keys to cycle through input history (PowerShell-style).
  • Toolbar — Contains the + (attach file), Ask button, ▲▼ navigation arrows, and Search box.
  • Response panel — Displays queries and AI responses. Each response has a toolbar with copy, thumbs up/down, and navigation buttons.

Sync Indicator

The small text next to your username in the header shows sync status:

  • ⟳ syncing — sync in progress
  • ✓ HH:MM — last successful sync time
  • ⚠ sync error — sync failed (check Google connection)
  • ⚠ offline — no network connection

4. Asking a Question

Mobius uses a two-step process for AI queries:

  • Step 1 — First Ask: Type your question and press Ask. Mobius prepares and displays a query package showing exactly what will be sent to the AI. Review it before sending.
  • Step 2 — Second Ask: Press Ask again to send the query to the AI and receive a response.
Ctrl+Enter can be used instead of clicking Ask. Press Escape to cancel a pending query or clear the input box.

Query Preview

The preview box shows the full query package: the model selected (ASK), instruction mode (INSTRUCTIONS), conversation history count (HISTORY), your query (QUERY), any attached files (FILES), and whether environment context is included (CONTEXT).

Input History

Use ↑ and ↓ arrow keys in the input box to cycle through your previous queries, similar to a terminal. History is saved across sessions.

5. Choosing an AI Model

Mobius remembers the last model you used. The default on first use is Groq. To switch models, use the Ask: prefix — this routes the current query and saves that model as your new default.

CommandRoutes to
Ask: GroqGroq — Llama 3.3 70B (fast, general)
Ask: GeminiGoogle Gemini 2.5 Flash (long context, vision)
Ask: MistralMistral Codestral (coding)
Ask: GitHubGPT-4o via GitHub Models
Ask: WebTavily web search + AI summary
Ask: QwenLocal Ollama — Qwen 2.5 Coder 7B (coding, private)
Ask: DeepSeekLocal Ollama — DeepSeek R1 7B (reasoning, private)
Ask: LocalLocal Ollama — default local model
Local models (Qwen, DeepSeek) run entirely on your device via Ollama. No data leaves your machine. Ollama must be running via start-ollama.bat on the laptop.

Automatic Fallback

If a model fails, Mobius automatically tries the next in the chain: Groq → Gemini → Mistral → GitHub. The response shows which model was actually used. A fallback does not change your saved default.

Example

Ask: Gemini. Summarise the key points of green infrastructure planning.

6. Attaching Files

  • Click the + button in the toolbar to open the file picker.
  • Supported types: images (jpg, png, gif, webp), PDF, text files (txt, md, csv, js, py, html, json).
  • Multiple files can be attached at once. Each appears as a chip — click ✕ to remove.
  • Images are automatically routed to Gemini for vision processing (OCR, analysis, description).
  • Non-image files are appended as text content to your query.
File uploads are processed server-side and not stored permanently — they exist only for the duration of your query.

7. Commands Reference

Commands are handled by Mobius before any AI is involved. They use a colon-prefix format: Command: argument. Single-word commands (Date, Time, Location, Device, Access, List, History, Status) work without a colon.

Access

Grants Mobius read access to a local folder on your device. Required before using Find or List.

  • A folder picker opens. Navigate to your root folder and click Select Folder.
  • Access is saved between sessions via IndexedDB. Run again to change folder.
  • Browser support: Chrome and Edge on desktop and Android only.

Ask:

Routes your query to a specific AI model and saves it as your new default.

  • Usage: Ask: modelname. your question
  • Without a model name, uses your last saved model.

See page 5 for the full model list.

Code:

Activates Code Mode — a structured coding assistant for software projects. See page 12 for full details.

Date

Displays the current date. Example output: Monday, 10 March 2026

Device

Displays detailed information about your current device and browser — OS, browser version, device type, screen resolution, CPU cores, RAM, GPU, storage, connection type, bandwidth, and touch support.

Find:

Searches for files and folders by name within the current local root folder.

Focus:

Loads a Google Drive file as persistent context for the current session. The file is attached to every AI query until you end the focus.

CommandWhat it does

If the file is not found in your Mobius Drive folder, Mobius searches your whole Drive. If still not found, it creates a new file in the Mobius folder.

Google:

Manages connected Google accounts. Mobius supports up to three accounts: personal, family, and work.

CommandWhat it does
Google: statusShow which accounts are connected
Google: connect personalConnect your personal Gmail account
Google: connect familyConnect your shared family Gmail account

Each account requires a separate OAuth sign-in. You will be redirected to Google and then returned to Mobius automatically.

History

Displays your past conversation sessions. Sessions are grouped by 30-minute gaps. Click a session to open it. Right-click for options: Open chat, Copy summary, Copy full chat, Copy as context.

List

Lists the top-level contents of the current local root folder. Runs Access first if no folder is granted.

Location

Displays your current location based on IP geolocation (city, region, country, timezone). Uses ipapi.co. VPNs will show the VPN location.

New

Starts a fresh chat session — clears the response panel and resets the conversation history. Optionally accepts a topic to start the new session with a query.

  • New — clear and start fresh
  • New: summarise this week's calendar — clear and immediately ask a question

Status

Runs a live health check of all Mobius systems and displays results in the response panel.

  • Server availability (local and Vercel)
  • Google account connections (all three labels)
  • Supabase / chat history
  • Ollama (local AI, desktop only)
  • Service Worker (PWA)
  • Current session state (code mode, focus file)

Sync:

Manually triggers a sync of your account data into the Mobius index files. Sync also runs automatically on login and every 5 minutes for Google accounts.

Time

Displays the current local time. Example output: 3:45 PM

8. Chat History

Mobius saves all conversations to Supabase. To browse past sessions, type History and press Ask.

Right-clicking a session gives four options:

  • Open chat — Loads the full session into the response panel and restores it as context for follow-up queries.
  • Copy summary — Copies a brief summary (first 3 exchanges) to the clipboard.
  • Copy full chat — Copies the entire session to the clipboard.
  • Copy as context — Attaches the session summary as a file chip, ready to send with your next query.
Sessions are grouped by 30-minute gaps. The session title is the first query of that session.

9. Searching the Response Panel

  • Use the Search box in the toolbar to highlight matching text anywhere in the current response panel.
  • Use the ▲▼ navigation arrows in the toolbar to jump between query/response pairs in the current session.
  • Each response also has its own ▲▼ buttons in the response toolbar for the same navigation.

10. Google Integration

Mobius connects to up to three Google accounts simultaneously, identified by label:

LabelAccount typeAccess
personalMain personal GmailCalendar, Email metadata, Drive
familyShared family GmailCalendar, Email metadata, Drive
workWork Gmail (boon.curtin)Calendar, Email metadata, Drive

Connecting accounts

Your personal account connects automatically on first login. To connect additional accounts:

  • Google: connect family — redirects to Google sign-in, then returns automatically
  • Google: connect work — same process for your work account
  • Google: status — check which accounts are connected

What Mobius reads

  • Calendars — upcoming events for the next 60 days (title, time, location)
  • Email — unread message metadata only (sender, subject, date — no content)
  • Drive — file listings (filename, modified date)

What Mobius never does

  • Never reads email content
  • Never writes to calendars, Gmail, or Drive
  • Never creates, modifies, or deletes any file outside the Mobius folder

The Mobius Drive folder

Mobius maintains a single folder called Mobius in your personal Google Drive. This is the only location Mobius writes to. It contains index files, code documents, and Focus working files.

Focus: command

Use Focus: filename to load a Google Drive file as persistent context for your session. See the Commands Reference (page 7) for full details.

11. Sync & Dropbox

How sync works

Mobius maintains four lightweight index files in your Mobius Drive folder. These are plain-text metadata files — no content, only identifiers. AI queries run against these local indexes rather than hitting live APIs each time.

Index fileContents
calendar.indexEvents from all 3 Google calendars — date, time, title, account label
email.indexUnread email metadata from all 3 inboxes — date, sender, subject, label
drive.indexFile listings from all 3 Google Drives — filename, modified date, label
dropbox.indexFull Dropbox file listing — filename, path, size, modified date

Sync schedule

  • On login — full sync of all Google accounts (automatic, runs in background)
  • Every 5 minutes — incremental Google sync while the tab is open (fetches only changes since last sync)
  • Dropbox — syncs on login and on demand only (Sync: dropbox)

Sync indicator

Watch the header for sync status: ⟳ syncing while active, ✓ HH:MM when done, ⚠ offline if unreachable.

Connecting Dropbox

Dropbox connects via OAuth the same way as Google. Once connected, run Sync: dropbox to build the initial index. Subsequent logins will sync it automatically.

Mobius has read-only access to Dropbox. It never writes to, modifies, or deletes any Dropbox file.

Manual sync commands

  • Sync: all — full refresh of all Google indexes
  • Sync: dropbox — refresh Dropbox index
  • Sync: status — show last synced timestamps for all sources

12. Code Mode

Code Mode is a structured coding assistant framework built into Mobius. It ensures AI always works from accurate, verified project context before generating or editing code.

Starting a session

Type Code: projectname and press Ask. A folder picker opens — navigate to the parent folder containing your project. Mobius finds the matching project folder and loads any existing .map, .repo, and .audit files from the project's documents/ subfolder.

The command pipeline

CommandWhat it doesOutput
Code: repoParses all JS/HTML files — functions, imports, exports, routesprojectname.repo
Code: mapSends repo + README + previous map to Gemini for an architectural summaryprojectname.map
Code: scannAI static analysis — no AI involved. Detects secrets, bare awaits, empty catches, missing res.ok, large functions, ENV mismatchesprojectname.scan
Code: audit newRuns repo + scan, asks Gemini for a briefing and ordered fix planprojectname.audit
Code: auditResumes an existing audit — re-scans, reloads context, continues with GeminiUpdated .audit
Code: audit endCloses the audit with a Gemini final summaryFinal .audit
Code: allRuns repo → map → audit in sequenceAll three files
Code: statusSaves session state snapshotprojectname.status
Code: showDisplays all loaded session content in the panelDisplay only
Code: endEnds the session—

Output files

All output files are saved to two places: the Mobius Google Drive folder (automatic) and a local download link. Save local downloads into the project's documents/ subfolder — Mobius opens the save dialog there automatically.

AI + nAI

Code Mode distinguishes between AI tools (Claude, Gemini — for architecture and novel logic) and nAI tools (ESLint, Prettier, Vitest — deterministic checks with no hallucinations). Code: scan is entirely nAI — it never calls an AI model.

Edit protocol

When asking AI to modify code, always request the FIND/REPLACE format to ensure changes are surgical:

FILE: filename.js
FIND: [exact existing code]
REPLACE: [new code]

Code Mode reads from your local file system. The project folder must be selected each session via the folder picker.

13. Environment Awareness

Mobius automatically detects and sends your environment context with every AI query. The AI is aware of:

  • Current date, time, timezone and UTC offset
  • Your location (city, region, country) via IP geolocation
  • Local currency
  • Operating system and browser version
  • Device type (desktop, tablet, mobile)
  • Screen resolution
  • Network connection type, bandwidth, and latency (where available)
  • Dark/light mode preference
  • Online status
This context is visible in the Mobius Query preview box before sending. It is sent as a virtual file attachment, not as part of your visible query text.

14. Tips

  • Use Ctrl+Enter to trigger Ask. Use Escape to cancel a pending query or clear the input box.
  • Use ↑/↓ arrow keys in the input box to cycle through your previous queries.
  • Single-word commands (Date, Time, Location, Device, Access, List, History, Status) work without a colon.
  • The query preview box shows exactly what Mobius will send — always check it before the second Ask.
  • Switch AI models with Ask: Gemini — Mobius remembers your choice for all future queries.
  • Use Ask: Qwen or Ask: DeepSeek for privacy-sensitive coding tasks — runs entirely on your device.
  • Access only needs to be run once per browser — permission is saved via IndexedDB. Run again to change root folder.
  • Use Find: report Ext: pdf From: last month to narrow file searches by type and date.
  • Use Copy as context from chat history to carry a previous session into a new query.
  • After connecting a Google account, run Sync: all to build fresh indexes immediately.
  • Run Google: status to check which accounts are connected, or Sync: status to see last sync times.
  • For coding sessions, run Code: all at the start of each session to get a fresh repo, map, and audit in one step.
  • Save Code Mode output files into your project's documents/ subfolder — Mobius opens the save dialog there automatically when a Code session is active.
