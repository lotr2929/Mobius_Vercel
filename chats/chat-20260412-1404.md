## What this is
GPRTool is a web-based application designed for Ground Penetrating Radar (GPR) data visualisation and analysis, featuring 3D scene rendering with Three.js. It utilises HTML, CSS, and JavaScript for the client-side, complemented by Python for back-end data processing, including an LAI pipeline.

## Current focus
The project is actively undergoing a significant refactoring effort to modularise the codebase. This involves extracting components from `index.html` into discrete ES modules within the `app/js/` directory, adhering to a defined target file structure.

## Key constraints
- **Browser Compatibility:** Development and testing must use Microsoft Edge due to known rendering and performance issues with Google Chrome.
- **File System Operations:** All file operations must be conducted using MCP filesystem tools; direct bash commands are forbidden.
- **Code Modularity:** New or modified client-side code must be extracted into ES modules within `app/js/`. The `index.html` file should eventually become a lean shell.
- **Language Separation:** Application code in `app/` is strictly HTML, CSS, and JavaScript. Python code is confined to `lai/` (LAI pipeline) and `_archive/backend/`, never deployed to the browser.
- **Architecture & Simplicity:** Solutions must be the simplest fit for existing logic and structure, respecting the `app/js/` target directory structure (only `lib/` and `tools/` subfolders are permitted).

## Known issues
- Chrome: GPRTool currently experiences rendering and performance issues in Google Chrome.
